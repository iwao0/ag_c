# Tokenizer C11準拠チェックリスト

本チェックリストは「Tokenizer観点」でのC11準拠状況を確認するためのものです。  
方針として、`strict C11` は**デフォルトOFF**（拡張許可）を維持します。

## 1. 方針確認
- [x] `strict C11` デフォルトOFFを維持する
- [x] strict ON時のみ拡張（例: `0b...`）を拒否する

## 2. 文字種・識別子
- [x] C11キーワード一式をトークン化できる
- [x] UCN (`\uXXXX`, `\UXXXXXXXX`) を識別子で受理できる
- [x] UCNのコードポイント妥当性チェックを行う

## 3. 文字列・文字定数
- [x] 文字列接頭辞 `L/u/U/u8` を受理できる
- [x] 文字定数接頭辞 `L/u/U` を受理できる
- [x] 隣接文字列リテラル連結に対応する（Parser側連結）
- [x] 接頭辞付きマルチ文字定数を実装定義として受理する
- [ ] 接頭辞付きマルチ文字定数の値形成規則をC11実装定義として明文化する（ドキュメント追記）

## 4. 数値リテラル
- [x] 10/8/16進整数リテラルを受理できる
- [x] 浮動小数点（10進/16進）を受理できる
- [x] 整数サフィックス (`U/L/LL`) を保持できる
- [x] 浮動小数点サフィックス (`f/F/l/L`) を保持できる
- [ ] `l/L` サフィックス（long double）の意味処理を実装する（現状はdouble経路）

## 5. 前処理寄り仕様
- [x] トライグラフ置換を設定で有効化できる
- [x] 行継続（`\\\n`）を処理できる
- [x] コメント（`//`, `/* */`）を処理できる

## 6. strict C11との境界
- [x] `0b...` を拡張扱いとして実装済み（strict時拒否）
- [ ] strict C11監査モード（拡張トークン検出ログ）の導入可否を検討する

## 7. テストと回帰
- [x] Tokenizer単体テストで主要機能をカバーする
- [x] 性能回帰チェック（case別）を運用している
- [x] 継続/停止ゲートで最適化劣化を検出できる
- [ ] C11準拠観点だけを抽出した回帰テストグループを追加する

# Cプリプロセッサ実装の振り返り (Phase 1)

## 実装内容
C言語のプリプロセッサ命令（`#include`, `#define`, `#if` 等）を実装するための中間処理基盤として `src/preprocess/preprocess.c` を作成しました。

### 1. `tokenizer.c` の拡張
プリプロセッサが `#include` などを認識するためには、行頭の `#` を特別に扱う必要があります。そのため、`tokenize()` の処理を拡張し、各トークンに対して以下の情報を付与するようにしました。
- `at_bol` (Beginning of Line): そのトークンが行の先頭にあるかどうか。
- `has_space`: そのトークンの直前に空白文字が存在したかどうか（関数マクロ呼び出し等の判定に必要）。

### 2. コンパイラパイプラインの変更
`main.c` の処理パイプラインを以下のように変更しました。
`tokenize()` (字句解析) -> **`preprocess()` (プリプロセス)** -> `program()` (構文解析)

### 3. ディレクティブ分離ロジックの導入
`preprocess.c` において、`at_bol` が true かつ `#` 記号であるトークン列を「ディレクティブ行」として通常のコードトークンと分離する基盤を実装しました。

## Phase 2: `#include` ディレクティブの実装

### 1. トークナイザの拡張
`#include <stdio.h>` などの中の `.` を正しく認識できるようにするため、`tokenizer.c` を更新して浮動小数点数でない `.` を1文字の記号として扱うようにしました。また、現在の `user_input` 状態を保存・復元できるように、ゲッター・セッターを追加しました。

### 2. ファイルの読み込みと再帰的なトークナイズ
`preprocess.c` 内に `read_file` 関数を実装し、指定されたファイルの中身をバッファとして読み込むようにしました。その後、以下のように処理をつなげています。
- 読み込んだ文字列に対して `tokenize()` を実行
- 分離したトークン列に対して再帰的に `preprocess()` を実行
- 帰ってきた新しいトークン列を、メインのトークン列につなぐ

### 3. テストの作成
`test_preprocess.c` を新設し、インクルードされるダミーのヘッダーファイルを動的に生成して `ag_c` に読み込ませ、期待されるAST・動作になるかを検証するE2Eテストを追加しました。

## Phase 3: オブジェクトマクロと定数置換 (`#define`, `#undef`)

### 1. マクロ定義の保持
`preprocess.c` に、連結リスト構造で `マクロ名` と `置換トークン列` のペアを保存するための `macro_t` と `macros` (保存リスト) を実装しました。
`#define IDENT ...` は以降の同一行のトークンをリストとしてコピー・保存するようにしました。
`#undef IDENT` は `macros` リストから対象を検索してリンクを外す処理として実装しました。

### 2. マクロの再帰的展開と hideset による無限ループ防止
通常の走査されるトークンが識別子であった場合、マクロとして定義されていれば展開する機能を実装しました。
C言語の標準規格に則り、マクロ展開による無限ループを防ぐため、トークン型 (`token_t`) に `hideset` （展開中マクロの記録構造）を追加しました。
あるマクロが展開される際、展開先の全トークンの hideset にそのマクロ名が追加されるため、元のマクロの中から再度自分自身を呼び出しても、そこで展開が停止するようになっています。
`hideset` に関して「名前をフリーした際、hideset 内に持たせていた参照先の文字列まで一緒に消えてしまう」というバグが生じましたが、ディープコピーを行うことで解決しました。

## Phase 4: 条件付き取り込み・条件分岐 (`#if`, `#ifdef`, `#ifndef`, `#else`, `#elif`, `#endif`)

### 1. `cond_incl` コンテキストの実装とスキップ処理
条件分岐におけるネストや `#else`, `#elif` などの状態を持つ構造体 `cond_incl_t` を導入し、連結リストによるスタック機構（現在いる最も内側のコンテキスト）を作りました。
`#if` などの条件が `false` だった場合や、既に `true` 分岐を通過して `#else` に到達した場合は、後続のトークン群をパースせずに `skip_cond_incl` ヘルパを用いて `#elif` / `#else` / `#endif` まで一気に読み飛ばす（無視する）処理を実装しました。

### 2. トークナイザの調整
`tokenizer.c` に `defined(X)` の評価に必要となる否定演算子 `!` や `~`、論理演算子 `&&`, `||` などの1文字・2文字演算子処理の拡張を行いました。

### 3. 定数式エバリュエータの実装
C言語コンパイラは `#if ...` の定数式におけるマクロを展開した上で計算等を行うため、以下のプロセスで評価を行う `evaluate_constexpr` を作成しました。
1. `#if` の行だけを分離して取得する。
2. その文字列の中の `defined(MACRO)` および `defined MACRO` を走査し、マクロが存在するかどうかで `1` または `0` トークンに強制置換する（マニュアルでの早期解決）。
3. その文字列トークンを `preprocess()` にかけ、通常マクロの展開を行う。
4. 新規に実装した専用の再帰下降パーサ (`const_expr` 群) を通して値を計算する（`+`, `-`, `*`, `/`, `==`, `!=`, `<`, `>`, `<=`, `>=`, `&&`, `||` など）。

これにより、`#if 1 + 2 == 3` や `#elif !defined(FOO)` といった複雑な条件付き取り込みに対応しました。

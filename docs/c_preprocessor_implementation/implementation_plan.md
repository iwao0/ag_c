# Cプリプロセッサ（CPP）の実装計画

C言語のプリプロセッサ命令（`#include`, `#define`, `#if` 等）およびマクロ展開機能を実装し、外部ヘッダファイルの読み込みや定数・インラインマクロを利用できるようにします。

## 設計方針

現状のコンパイラは `文字列` -> `トークン列 (tokenizer.c)` -> `AST (parser.c)` の順に処理を行っています。
プリプロセッサは字句解析と構文解析の中間に別フェーズとして組み込みます。

**処理フロー:**
`文字列` -> `tokenize()` -> **`preprocess() (ここを新設)`** -> `program()` -> `gen()`

## 対象とする主な機能

1. **ファイルインクルード**: `#include "file.h"`, `#include <file.h>`
2. **オブジェクトマクロ**: `#define PI 3.14`, `#undef PI`
3. **関数マクロ**: `#define MAX(a,b) ((a)>(b)?(a):(b))`
4. **条件分岐**: `#if`, `#ifdef`, `#ifndef`, `#else`, `#elif`, `#endif`
5. **その他**: `#error`, `#line` 等のディレクティブや、文字列化演算子 `#`、トークン結合演算子 `##` のサポート。

---

## 実装ステップ

### 1. プリプロセッサ基盤の開発 (`preprocess/preprocess.c`)
- 新しいソースファイル `src/preprocess/preprocess.c` を作成。
- `token_t *preprocess(token_t *tok)` を定義し、マクロ展開やディレクティブを処理して新しい（展開済みの）トークン列を返す構造を作る。
- 行の先頭にある `#` から始まるディレクティブ行を認識・分離する処理。

### 2. `#include` ディレクティブの実装
- 指定されたパス（カレントディレクトリや、システムインクルードパス）からヘッダファイルを読み込む関数を実装。
- 読み込んだファイルの中身を `tokenize()` にかけ、現在のトークン列の `#include` の位置に結合（マージ）する。

### 3. オブジェクトマクロと単純な展開 (`#define`, `#undef`)
- マクロ名と置換トークン列を紐づける環境（ハッシュマップまたは連結リストベースの辞書）を実装。
- トークン列を走査し、識別子がマクロとして登録されていればその展開結果でトークンを置き換える機能。
- 再帰的なマクロ展開と、無限ループ防止の仕組み（展開済みマクロを追跡するためのフラグ `hideset` など）の導入。

### 4. 条件付き取り込み (`#if`, `#ifdef`, `#ifndef`)
- 非アクティブなブロック（条件が満たされないブロック）内のトークンを `#else` や `#endif` まで読み飛ばす機能（現在の `#if` のネスト状態をスタックで管理）。
- `#if` 行の定数式を評価し、真理値を判定するためのプリプロセッサ専用のミニパーサ、または定数式エバリュエータの実装。

### 5. 関数マクロ (`#define MACRO(a, b)`)
- パラメータ付き `#define` のパース処理。
- マクロ呼び出し時に実引数トークン列を収集し、定義された置換リスト内の仮引数を実引数で置き換える展開機能。
- `##` (トークン結合) および `#` (文字列化) 演算子の実装。

## 検証計画

- 専用の `test_preprocess.c` または E2E テストファイル群を用意。
- `#define` の置き換えが正確に行われるか、`#include` で別ファイルの変数が読めるかをテスト。
- Cの標準規格（C11等）に沿った複雑なマクロ展開規則をパスできるか多様なテストケース（例: `#define a(x) x`, `a(1)`）を作成して自動検証。
